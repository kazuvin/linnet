# ペルソナ定義

## Persona 1: タクミ（22歳）- 学び始めのギタリスト

### プロフィール

- **ギター歴**: 1年半。コード弾きはできるが、理論は独学で断片的
- **環境**: 自宅でアコギ。スマホかノートPCで作業
- **音楽**: 弾き語り系のシンガーソングライターに憧れている

### 課題・ペインポイント

- 「このキーで使えるコードは何？」がすぐに出てこない
- モーダルインターチェンジという言葉は聞いたことがあるが、実際にどう使うかわからない
- コード名は知っていても、指板上のどのポジションで押さえるかバリエーションがない
- YouTube や教則本の情報がバラバラで、一箇所にまとまったリファレンスが欲しい

### Linnet に求めること

- キーを選んだら使えるコードが一覧で見えること
- コードをタップしたら指板上の押さえ方がすぐわかること
- 難しい用語があっても視覚的に理解できるUI
- スマホでも快適に操作できること

### ゴール

「理論を頭で覚えるのではなく、触って試して感覚的に理解したい」

## Persona 2: ユウキ（28歳）- バンドの作曲担当

### プロフィール

- **ギター歴**: 8年。バンドでリードギター兼作曲担当
- **環境**: DAW（Logic Pro）で宅録。デスクトップPCメイン、外出時はスマホ
- **音楽**: ロック・オルタナ。王道進行に飽きてきた

### 課題・ペインポイント

- コード進行のアイデアがマンネリ化している
- モーダルインターチェンジやセカンダリードミナントは知っているが、使いこなせていない
- 「このコードの次に何が来たら面白いか」を素早く試したい
- 指板上でスケールやコードトーンを俯瞰して、リフやソロのアイデアに繋げたい

### Linnet に求めること

- コード進行をサッと組み替えて試行錯誤できるドラッグ&ドロップUI
- ダイアトニック以外のコード（モーダルインターチェンジ）が一目でわかる分類
- コード選択に連動した指板上のスケール/コードトーン表示
- 操作にもたつかない、レスポンスの速いインタラクション

### ゴール

「頭の中のアイデアを最速でコード進行に落とし込み、指板上で確認したい」

## Persona 3: マリ（35歳）- ギター講師

### プロフィール

- **ギター歴**: 18年。音楽教室でギターとコード理論を教えている
- **環境**: レッスンではタブレット（iPad）を使用。自宅ではPC
- **音楽**: ジャズ・ポップス。生徒のレベルに合わせて幅広いジャンルに対応

### 課題・ペインポイント

- レッスン中に「このキーのダイアトニックコードは...」と板書するのが非効率
- 生徒に指板上のコードの構成音やスケールを視覚的に説明するツールが欲しい
- 既存のアプリは情報過多で、初心者の生徒が混乱する
- モーダルインターチェンジの説明時に「なぜこのコードが使えるか」を示したい

### Linnet に求めること

- レッスン中にサッと見せられるクリアな指板ビジュアライゼーション
- 段階的に情報量を調整できるUI（初心者向け → 上級者向け）
- タブレットのタッチ操作でスムーズに使えること
- コードの機能（トニック・サブドミナント・ドミナントなど）が視覚的にわかる表現

### ゴール

「生徒の理解度に合わせて、コード理論を直感的に見せるツールが欲しい」

## ペルソナから導かれるデザイン要件

| 要件              | タクミ               | ユウキ       | マリ                           |
| ----------------- | -------------------- | ------------ | ------------------------------ |
| モバイル対応      | 必須                 | あると便利   | 必須（タブレット）             |
| 指板ビジュアル    | 学習用               | 確認・発想用 | 教材用                         |
| 情報量の調整      | 少なめが良い         | 多くてOK     | 切り替えたい                   |
| ドラッグ&ドロップ | 使えると楽しい       | 必須         | あると便利                     |
| レスポンス速度    | 普通でOK             | 速さ重視     | サッと見せたい                 |
| 理論用語の補助    | ツールチップ等が必要 | 不要         | 生徒に見せる時は非表示にしたい |

### 共通のデザイン原則

1. **指板が主役**: ギター指板のビジュアライゼーションがアプリの中心。大きく、見やすく
2. **段階的な情報開示**: 必要な情報だけを表示し、詳細は掘り下げられる（Progressive Disclosure）
3. **タッチフレンドリー**: モバイル・タブレットでもストレスなく操作できるタッチターゲットサイズ
4. **速いフィードバック**: コード選択 → 指板表示の切り替えは即座に。待たせない
5. **視覚的な区別**: ダイアトニック vs モーダルインターチェンジなど、種類の違いが色やアイコンで直感的にわかる
